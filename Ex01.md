## What-if Scenario & Multiple Options

## 1. Phân tích ngắn gọn bối cảnh

SpeedyCart đang gặp vấn đề hiệu năng ở chức năng giỏ hàng tạm thời. Đây là khu vực có tần suất đọc/ghi rất lớn vì người dùng liên tục thêm sản phẩm, xóa sản phẩm hoặc thay đổi số lượng. Nếu mọi thao tác đều ghi trực tiếp xuống SQL Database, hệ thống dễ bị nghẽn, tăng độ trễ và làm giảm trải nghiệm người dùng.

Vì vậy, cần thiết kế prompt đa chiều để AI không chỉ đưa ra một giải pháp duy nhất, mà phải phân tích nhiều phương án lưu trữ khác nhau, so sánh trade-off, đánh giá tình huống lỗi như Redis crash, đồng thời đề xuất cấu hình triển khai thực tế bằng Java Spring Boot.

Redis phù hợp với bài toán giỏ hàng tạm thời vì có tốc độ đọc/ghi cao nhờ lưu trữ in-memory. Tuy nhiên, Redis vẫn cần cơ chế persistence như RDB hoặc AOF để giảm rủi ro mất dữ liệu khi crash. Theo tài liệu Redis, RDB tạo snapshot dữ liệu theo thời điểm, còn AOF ghi lại các thao tác ghi để có thể replay khi Redis khởi động lại. ([Redis][1]) Spring Data Redis cũng hỗ trợ cấu hình `RedisTemplate` để thao tác với Redis trong ứng dụng Spring Boot. ([Home][2])

### 1. Tổng quan bài toán

Trong hệ thống thương mại điện tử như SpeedyCart, giỏ hàng tạm thời là dữ liệu có tần suất thay đổi rất cao. Người dùng có thể thêm sản phẩm, xóa sản phẩm, cập nhật số lượng, áp mã giảm giá hoặc quay lại giỏ hàng nhiều lần trước khi checkout.

Nếu toàn bộ thao tác này được ghi trực tiếp vào SQL Database, hệ thống sẽ gặp các vấn đề:

- Số lượng query ghi tăng mạnh.
- Database bị quá tải ở các giờ cao điểm.
- Độ trễ khi thao tác giỏ hàng tăng.
- Khả năng mở rộng kém.
- Những dữ liệu tạm thời lại chiếm nhiều tài nguyên của database chính.

Vì vậy, cần xem xét nhiều phương án lưu trữ khác nhau và chọn giải pháp cân bằng giữa tốc độ, độ bền dữ liệu, chi phí triển khai và khả năng phục hồi khi có lỗi.

## 2. Các phương án lưu trữ giỏ hàng tạm thời

Có thể xem xét 4 phương án chính:

1. Lưu trực tiếp vào SQL Database.
2. Lưu vào Redis In-memory Cache.
3. Lưu ở Client-side Session/Cookie.
4. Mô hình Hybrid: Redis làm primary cart storage, SQL làm fallback/persistent backup.

## 4. Phân tích từng phương án

### 4.1. SQL Database

SQL Database có ưu điểm là dữ liệu bền vững, dễ truy vấn, có transaction và đảm bảo tính nhất quán tốt. Tuy nhiên, với giỏ hàng tạm thời, việc ghi quá thường xuyên vào SQL sẽ tạo áp lực lớn cho database.

Ví dụ, một người dùng chỉ thay đổi số lượng sản phẩm từ 1 lên 2 cũng tạo một transaction ghi. Nếu có hàng trăm nghìn người dùng cùng thao tác, database sẽ bị nghẽn.

SQL nên được dùng cho:

- Đơn hàng đã checkout.
- Lịch sử thanh toán.
- Thông tin sản phẩm chính thức.
- Dữ liệu cần audit.
- Dữ liệu backup/fallback của giỏ hàng.

Không nên dùng SQL làm nơi xử lý chính cho mọi thao tác giỏ hàng tạm thời trong hệ thống tải lớn.

---

### 4.2. Redis In-memory Cache

Redis là lựa chọn rất phù hợp cho giỏ hàng tạm thời vì dữ liệu được lưu trong bộ nhớ, tốc độ đọc/ghi rất cao. Mỗi giỏ hàng có thể lưu theo key như:

```text
cart:{userId}
cart:{anonymousSessionId}
```

Ví dụ dữ liệu:

```json
{
  "userId": 1001,
  "items": [
    {
      "productId": 501,
      "quantity": 2,
      "price": 120000
    },
    {
      "productId": 888,
      "quantity": 1,
      "price": 99000
    }
  ],
  "updatedAt": "2026-06-23T14:00:00"
}
```

Có thể đặt TTL cho giỏ hàng:

```text
cart TTL = 7 ngày hoặc 30 ngày
```

Ưu điểm:

- Tốc độ rất nhanh.
- Giảm tải cho SQL Database.
- Dễ đặt TTL để tự động xóa giỏ hàng cũ.
- Phù hợp với dữ liệu tạm thời.

Nhược điểm:

- Nếu Redis crash mà không bật persistence, dữ liệu có thể mất.
- Cần thiết kế cơ chế backup/fallback.
- Cần giám sát memory, eviction policy, cluster failover.

---

### 4.3. Client-side Session/Cookie

Với phương án này, dữ liệu giỏ hàng được lưu ở trình duyệt người dùng thông qua cookie, localStorage hoặc sessionStorage.

Ưu điểm:

- Không tạo tải lên server.
- Tốc độ nhanh.
- Dễ triển khai cho khách chưa đăng nhập.

Nhược điểm:

- Dữ liệu có thể bị người dùng chỉnh sửa.
- Cookie có giới hạn kích thước.
- Không phù hợp để lưu thông tin nhạy cảm như giá sản phẩm chính thức.
- Khi người dùng đổi thiết bị, dữ liệu không đồng bộ.
- Khi người dùng xóa trình duyệt hoặc cookie, giỏ hàng biến mất.

Phương án này chỉ nên dùng cho giỏ hàng đơn giản hoặc làm lớp tạm thời cho người dùng chưa đăng nhập.

---

### 4.4. Hybrid Redis + SQL

Đây là phương án phù hợp nhất cho SpeedyCart.

Redis xử lý các thao tác đọc/ghi nhanh của giỏ hàng. SQL Database chỉ lưu dữ liệu cần bền vững hoặc dùng làm fallback khi Redis gặp sự cố.

Luồng đề xuất:

1. Khi người dùng thêm/sửa/xóa sản phẩm:
   - Ghi vào Redis trước.
   - Đặt TTL cho cart.
   - Có thể ghi async xuống SQL theo chu kỳ hoặc khi có thay đổi quan trọng.

2. Khi Redis lỗi:
   - Ghi tạm thao tác cart xuống SQL fallback table.
   - Đánh dấu trạng thái `redis_unavailable`.
   - Khi Redis hoạt động lại, đồng bộ dữ liệu từ SQL fallback về Redis.

3. Khi người dùng checkout:
   - Đọc cart từ Redis.
   - Kiểm tra lại giá và tồn kho từ SQL.
   - Tạo order chính thức trong SQL.
   - Xóa cart khỏi Redis sau khi checkout thành công.

## 5. What-if Scenario: Redis crash hoặc mất nguồn

### 5.1. Điều gì xảy ra nếu Redis crash?

Nếu Redis chỉ chạy in-memory và không bật persistence, toàn bộ dữ liệu giỏ hàng trong bộ nhớ có thể bị mất khi Redis crash hoặc mất nguồn.

Hậu quả:

- Người dùng quay lại trang và thấy giỏ hàng trống.
- Tỷ lệ chuyển đổi mua hàng giảm.
- Trải nghiệm người dùng kém.
- Có thể phát sinh khiếu nại nếu người dùng đã chọn nhiều sản phẩm.

---

## 5.2. Redis Persistence RDB

RDB là cơ chế snapshot dữ liệu Redis tại một thời điểm nhất định. Ví dụ Redis có thể lưu snapshot sau mỗi 5 phút hoặc sau một số lượng thay đổi nhất định.

Ví dụ cấu hình:

```conf
save 900 1
save 300 10
save 60 10000
```

Ý nghĩa:

- Sau 900 giây nếu có ít nhất 1 thay đổi thì snapshot.
- Sau 300 giây nếu có ít nhất 10 thay đổi thì snapshot.
- Sau 60 giây nếu có ít nhất 10000 thay đổi thì snapshot.

Ưu điểm:

- File nhỏ gọn.
- Khôi phục nhanh.
- Phù hợp cho backup định kỳ.

Nhược điểm:

- Có thể mất dữ liệu giữa hai lần snapshot.
- Nếu Redis crash trước khi snapshot, dữ liệu mới nhất chưa chắc được lưu.

---

## 5.3. Redis Persistence AOF

AOF ghi lại từng thao tác ghi vào Redis. Khi Redis khởi động lại, Redis có thể replay các thao tác này để tái tạo dữ liệu.

Ví dụ cấu hình:

```conf
appendonly yes
appendfsync everysec
```

Ý nghĩa:

- Bật AOF.
- Ghi dữ liệu xuống disk mỗi giây một lần.

Ưu điểm:

- Giảm nguy cơ mất dữ liệu.
- Phù hợp hơn RDB nếu cần độ bền dữ liệu cao.
- Có thể chỉ mất tối đa khoảng 1 giây dữ liệu nếu dùng `appendfsync everysec`.

Nhược điểm:

- File AOF thường lớn hơn RDB.
- Khôi phục có thể chậm hơn RDB.
- Tăng chi phí ghi disk.

---

## 5.4. Kết hợp RDB + AOF

Với giỏ hàng thương mại điện tử, nên bật cả RDB và AOF.

Cấu hình gợi ý:

```conf
save 900 1
save 300 10
save 60 10000

appendonly yes
appendfsync everysec
aof-use-rdb-preamble yes
```

Lý do:

- RDB giúp backup nhanh và khôi phục nhanh.
- AOF giúp giảm nguy cơ mất dữ liệu gần nhất.
- Kết hợp cả hai giúp cân bằng giữa hiệu năng và độ bền dữ liệu.

---

## 5.5. Fallback ghi tạm xuống SQL khi Redis lỗi

Khi Redis không khả dụng, hệ thống không nên trả lỗi ngay cho người dùng. Thay vào đó, có thể dùng fallback table trong SQL.

Ví dụ bảng:

```sql
CREATE TABLE cart_fallback (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    cart_data JSON NOT NULL,
    status VARCHAR(50) DEFAULT 'PENDING_SYNC',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Luồng xử lý:

1. Người dùng cập nhật giỏ hàng.
2. Service thử ghi vào Redis.
3. Nếu Redis thành công:
   - Trả kết quả thành công.

4. Nếu Redis lỗi:
   - Ghi dữ liệu cart vào bảng `cart_fallback`.
   - Trả kết quả thành công có cảnh báo nội bộ.
   - Ghi log để đội vận hành biết Redis đang lỗi.

5. Khi Redis hoạt động trở lại:
   - Job đồng bộ đọc các bản ghi `PENDING_SYNC`.
   - Ghi lại vào Redis.
   - Cập nhật trạng thái thành `SYNCED`.

## 6. Kiến trúc đề xuất cho SpeedyCart

### 6.1. Thành phần chính

```text
Client
  |
  v
Cart API Service
  |
  |-- Redis Cluster: lưu giỏ hàng tạm thời
  |
  |-- SQL Database: lưu order, user, product, cart fallback
  |
  |-- Background Sync Job: đồng bộ fallback SQL về Redis
```

---

### 6.2. Dữ liệu lưu trong Redis

Redis lưu dữ liệu giỏ hàng tạm thời:

```text
Key: cart:{userId}
Value: Cart object JSON
TTL: 7 ngày hoặc 30 ngày
```

Ví dụ:

```json
{
  "userId": 1001,
  "items": [
    {
      "productId": 10,
      "quantity": 2
    }
  ],
  "updatedAt": "2026-06-23T14:00:00"
}
```

Lưu ý: Không nên tin hoàn toàn giá ở Redis. Khi checkout, hệ thống phải kiểm tra lại giá chính thức từ SQL.

---

### 6.3. Dữ liệu lưu trong SQL

SQL lưu:

- User.
- Product.
- Inventory.
- Order.
- Payment.
- Cart fallback khi Redis lỗi.
- Cart snapshot nếu doanh nghiệp muốn phục hồi tốt hơn.

---

### 6.4. TTL cho giỏ hàng

Gợi ý TTL:

```text
Khách chưa đăng nhập: 1 - 7 ngày
Khách đã đăng nhập: 7 - 30 ngày
```

Ví dụ:

```text
cart:user:1001 TTL 30 ngày
cart:guest:session123 TTL 7 ngày
```

---

### 6.5. Luồng thêm/sửa/xóa sản phẩm

```text
1. Client gửi request thêm/sửa/xóa sản phẩm.
2. Cart API kiểm tra productId, quantity.
3. Cart API lấy cart hiện tại từ Redis.
4. Nếu chưa có cart, tạo cart mới.
5. Cập nhật item trong cart.
6. Ghi lại cart vào Redis với TTL.
7. Nếu Redis lỗi, ghi cart vào SQL fallback.
8. Trả response cho client.
```

---

### 6.6. Luồng checkout

```text
1. Client nhấn checkout.
2. Cart API đọc cart từ Redis.
3. Nếu Redis không có dữ liệu, kiểm tra SQL fallback.
4. Kiểm tra lại giá sản phẩm từ SQL.
5. Kiểm tra tồn kho.
6. Tính lại tổng tiền.
7. Tạo order trong SQL bằng transaction.
8. Trừ tồn kho.
9. Xóa cart khỏi Redis.
10. Cập nhật cart fallback nếu có.
```

# 7. Cấu hình RedisTemplate Java Spring Boot

## 7.1. application.yml

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password: ${REDIS_PASSWORD:}
      timeout: 3000ms
      lettuce:
```

```java
package com.speedycart.config;

import com.fasterxml.jackson.annotation.JsonTypeInfo;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.jsontype.BasicPolymorphicTypeValidator;
import io.lettuce.core.ClientOptions;
import io.lettuce.core.SocketOptions;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.autoconfigure.data.redis.RedisProperties;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.connection.RedisPassword;
import org.springframework.data.redis.connection.RedisStandaloneConfiguration;
import org.springframework.data.redis.connection.lettuce.LettuceClientConfiguration;
import org.springframework.data.redis.connection.lettuce.LettuceConnectionFactory;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.data.redis.serializer.GenericJackson2JsonRedisSerializer;
import org.springframework.data.redis.serializer.StringRedisSerializer;

import java.time.Duration;

@Configuration
public class RedisConfig {

    @Value("${spring.data.redis.host}")
    private String redisHost;

    @Value("${spring.data.redis.port}")
    private int redisPort;

    @Value("${spring.data.redis.password:}")
    private String redisPassword;

    @Value("${spring.data.redis.timeout:3000ms}")
    private Duration timeout;

    @Bean
    public RedisConnectionFactory redisConnectionFactory() {
        RedisStandaloneConfiguration serverConfig = new RedisStandaloneConfiguration();
        serverConfig.setHostName(redisHost);
        serverConfig.setPort(redisPort);

        if (redisPassword != null && !redisPassword.isBlank()) {
            serverConfig.setPassword(RedisPassword.of(redisPassword));
        }

        SocketOptions socketOptions = SocketOptions.builder()
                .connectTimeout(timeout)
                .build();

        ClientOptions clientOptions = ClientOptions.builder()
                .socketOptions(socketOptions)
                .autoReconnect(true)
                .build();

        LettuceClientConfiguration clientConfig = LettuceClientConfiguration.builder()
                .commandTimeout(timeout)
                .clientOptions(clientOptions)
                .build();

        return new LettuceConnectionFactory(serverConfig, clientConfig);
    }

    @Bean
    public RedisTemplate<String, Object> redisTemplate(
            RedisConnectionFactory redisConnectionFactory
    ) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(redisConnectionFactory);

        StringRedisSerializer stringSerializer = new StringRedisSerializer();

        ObjectMapper objectMapper = new ObjectMapper();

        BasicPolymorphicTypeValidator typeValidator =
                BasicPolymorphicTypeValidator.builder()
                        .allowIfSubType("com.speedycart")
                        .allowIfSubType("java.util")
                        .build();

        objectMapper.activateDefaultTyping(
                typeValidator,
                ObjectMapper.DefaultTyping.NON_FINAL,
                JsonTypeInfo.As.PROPERTY
        );

        GenericJackson2JsonRedisSerializer jsonSerializer =
                new GenericJackson2JsonRedisSerializer(objectMapper);

        template.setKeySerializer(stringSerializer);
        template.setHashKeySerializer(stringSerializer);

        template.setValueSerializer(jsonSerializer);
        template.setHashValueSerializer(jsonSerializer);

        template.afterPropertiesSet();

        return template;
    }
}
```

## 7.5. CartRedisService.java

```java
package com.speedycart.cart.service;

import com.speedycart.cart.dto.Cart;
import com.speedycart.cart.dto.CartItem;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.dao.DataAccessException;
import org.springframework.data.redis.core.RedisTemplate;
import org.springframework.stereotype.Service;

import java.time.Duration;
import java.time.LocalDateTime;
import java.util.Optional;

@Service
public class CartRedisService {

    private final RedisTemplate<String, Object> redisTemplate;
    private final Duration cartTtl;

    public CartRedisService(
            RedisTemplate<String, Object> redisTemplate,
            @Value("${app.cart.ttl-days:30}") long ttlDays
    ) {
        this.redisTemplate = redisTemplate;
        this.cartTtl = Duration.ofDays(ttlDays);
    }

    public Optional<Cart> getCart(Long userId) {
        try {
            Object value = redisTemplate.opsForValue().get(buildCartKey(userId));

            if (value instanceof Cart cart) {
                return Optional.of(cart);
            }

            return Optional.empty();
        } catch (DataAccessException ex) {
            // Có thể ghi log và chuyển sang đọc SQL fallback
            return Optional.empty();
        }
    }

    public void addOrUpdateItem(Long userId, Long productId, Integer quantity) {
        if (quantity == null || quantity <= 0) {
            throw new IllegalArgumentException("Quantity must be greater than zero");
        }

        Cart cart = getCart(userId).orElse(new Cart(userId));

        boolean existed = false;

        for (CartItem item : cart.getItems()) {
            if (item.getProductId().equals(productId)) {
                item.setQuantity(quantity);
                existed = true;
                break;
            }
        }

        if (!existed) {
            cart.getItems().add(new CartItem(productId, quantity));
        }

        cart.setUpdatedAt(LocalDateTime.now());

        saveCart(cart);
    }

    public void removeItem(Long userId, Long productId) {
        Cart cart = getCart(userId).orElse(new Cart(userId));

        cart.getItems().removeIf(item -> item.getProductId().equals(productId));
        cart.setUpdatedAt(LocalDateTime.now());

        saveCart(cart);
    }

    public void saveCart(Cart cart) {
        try {
            redisTemplate.opsForValue().set(
                    buildCartKey(cart.getUserId()),
                    cart,
                    cartTtl
            );
        } catch (DataAccessException ex) {
            // Nếu Redis lỗi, service tầng trên nên gọi CartFallbackService để ghi SQL
            throw ex;
        }
    }

    public void deleteCart(Long userId) {
        redisTemplate.delete(buildCartKey(userId));
    }

    private String buildCartKey(Long userId) {
        return "cart:user:" + userId;
    }
}
```

## 7.7. CartApplicationService.java

```java
package com.speedycart.cart.service;

import com.speedycart.cart.dto.Cart;
import org.springframework.dao.DataAccessException;
import org.springframework.stereotype.Service;

@Service
public class CartApplicationService {

    private final CartRedisService cartRedisService;
    private final CartFallbackService cartFallbackService;

    public CartApplicationService(
            CartRedisService cartRedisService,
            CartFallbackService cartFallbackService
    ) {
        this.cartRedisService = cartRedisService;
        this.cartFallbackService = cartFallbackService;
    }

    public void addOrUpdateItem(Long userId, Long productId, Integer quantity) {
        try {
            cartRedisService.addOrUpdateItem(userId, productId, quantity);
        } catch (DataAccessException ex) {
            Cart fallbackCart = cartFallbackService.getFromSqlFallback(userId);

            if (fallbackCart == null) {
                fallbackCart = new Cart(userId);
            }

            fallbackCart.getItems().removeIf(
                    item -> item.getProductId().equals(productId)
            );

            fallbackCart.getItems().add(
                    new com.speedycart.cart.dto.CartItem(productId, quantity)
            );

            cartFallbackService.saveToSqlFallback(fallbackCart);
        }
    }

    public void removeItem(Long userId, Long productId) {
        try {
            cartRedisService.removeItem(userId, productId);
        } catch (DataAccessException ex) {
            Cart fallbackCart = cartFallbackService.getFromSqlFallback(userId);

            if (fallbackCart != null) {
                fallbackCart.getItems().removeIf(
                        item -> item.getProductId().equals(productId)
                );

                cartFallbackService.saveToSqlFallback(fallbackCart);
            }
        }
    }
}
```
